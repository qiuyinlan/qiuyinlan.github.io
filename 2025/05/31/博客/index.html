这个 Token 可以用来做什么？

具体能干嘛，取决于你在创建 token 时勾选了哪些权限。常见用途如下：

1. Hexo 或其他博客自动部署

  • 把本地生成的静态页面自动上传到 GitHub Pages 仓库。

  • 你在 hexo deploy 的配置里写入这个 token，就可以用命令一键部署。

2. GitHub Actions 等自动化脚本

  • 自动运行 CI/CD（持续集成/部署）流程。

  • 推送代码、发布 Release、更新文件等操作无需手动登录。

3. 命令行操作 GitHub

  • 用 curl、gh 工具、Git 客户端等命令行工具执行 GitHub API 操作。

  • 适用于需要自动化控制 GitHub 仓库的场景。

4. 访问私有仓库

  • 如果你有私有仓库，token 允许你访问或修改这些仓库。

博客知识

  • hexo server（或简写 hexo s）是Hexo 自带的本地预览命令。

  • 它会启动一个本地的 Web 服务器（默认是//localhost:4000），把你的博客内容渲染成网页，方便你在本地电脑上预览博客效果。

  • 这个预览的内容是你本地的博客文件生成的“静态网页”，并没有发布到线上网站。

博客更新流程

  1. 编辑文章和内容（在 Obsidian 或 Hexo 目录）

  2. 运行脚本把内容同步到 Hexo 博客项目（同步 Markdown + 图片）

  3. 运行 hexo server 本地预览，确认没问题

  4. 运行 hexo generate（简写 hexo g）生成静态网页文件

  5. 运行 hexo deploy（简写 hexo d）把生成好的网页上传到 GitHub Pages 或你的远程服务器

    • 也可以通过 git push 把 Hexo 项目推送到 GitHub，然后 GitHub Actions 自动帮你部署。
